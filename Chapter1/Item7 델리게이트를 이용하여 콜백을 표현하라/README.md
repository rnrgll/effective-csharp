## 7. 델리게이트를 이용하여 콜백을 표현하라

### 콜백(Callback) 함수

* 동작의 수행 시점을 호출자가 아닌 외부에 위임하는 함수

  (특정 작업이 완료되었을 때 / 특정 조건이 충족되었을 때 외부에서 호출하도록 등록된 함수)

* C#에서는 델리게이트를 통해 타입 안정적인 콜백을 구현한다.

> #### 📝 인터페이스 vs 델리게이트
>
> 둘 모두 클래스 간의 상호 통신이 필요할 때 사용할 수 있다.
> - 인터페이스
>   - 클래스 간의 결합도가 상대적으로 높으며, 상속 구조를 강제한다.
> - 델리게이트
>   - 특정 메서드 시그니처만 맞으면 어떤 클래스의 메서드든(정적/인스턴스 무관) 연결할 수 있다.
>   - 런타임 시 통지 대상을 설정할 수 있고, 다수의 대상에게 통지를 보낼 수도 있다. 

* 활용

  * 비동기 처리 (서버가 클라이언트에게 비동기로 피드백을 줄 때)

  * 이벤트 응답, 사용자 입력 처리 등

 

### 델리게이트 (Delegate)

* 델리게이트란?

  * 특정 시그니처를 가진 메서드를 참조하는 타입

  * 메서드를 변수처럼 전달하거나 저장하는 데 사용된다.

* C#의 표준 델리게이트

  : `.NET` 프레임워크는 범용적으로 사용할 수 있는 표준 델리게이트를 제공한다.

  <figure><p>📝 <code>LINQ</code>는 델리게이트를 기반으로 구현됐다.</p></figure>

* C# 델리게이트 내부 구조

  * 클래스 기반 구조로, 모든 델리게이트는 `MulticastDelegate` 를 상속한다.

  * `MulticastDelegate` 클래스는 내부적으로 Invocation list를 유지한다.

  * 델리게이트에 함수를 추가(combine)하거나 제거(remove)하면 해당 리스트가 조작된다.

 

### 멀티캐스트 델리게이트 (Multicast Delegate)

* 하나의 델리게이트에 여러 메서드를 체인처럼 연결하여 한 번의 호출로 순차 실행할 수 있다.

  ```csharp
  Action notify = ShowMessage;
  notify += LogMessage; 
  
  notify(); // ShowMessage -> LogMessage 순차 실행
  
  ```

* 주의할 점

  * 예외(Exception)에 취약하다.

    : 체인 중간에 있는 메서드에서 예외가 발생하면, 그 즉시 실행이 중단되어 나머지 메서드들은 호출되지 않는다.

  * 반환값 소실

    : 반환값이 있는 델리게이트(`Func` 등)를 멀티캐스트로 사용하면, 마지막으로 호출된 메서드의 반환값만 남고 이전 값들은 모두 무시된다.

  * 델리게이트는 참조하고 있는 메서드를 계속 보유하기 때문에, 적절히 해제하지 않으면 GC가 수거하지 못해 메모리 누수가 발생할 수 있다.

* ✅ 해결 방법

  * 수동 순회 코드를 작성해, 직접 호출 과정을 제어해야 한다.
